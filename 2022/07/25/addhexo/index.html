搭建hexo博客系统并上传Github进行托管-基础建站篇

发表于 2022-07-25 更新于 2022-07-27 分类于 Hexo Valine：
Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

环境部署

1、安装Node.js

由于Hexo博客系统是基于Node.js的，所以我们搭建Hexo需要先安装Node.js。

在Node.js官网下载长期支持版，然后打开下载好的安装程序，安装选项全部选择默认就好了,接下来一直点击Next下一步。

安装好以后打开cmd输入node -v，如果显示的是下载的版本号就说明安装成功。

2、安装Git

官网地址：git - 快速版本控制

下载完成后打开安装程序，安装选项全部默认，剩下的交给next。

安装完成后打开cmd输入git --version检查是否安装成功，如果显示的是下载时的版本号就说明安装成功。

安装hexo

在一个你感觉合适的地方创建一个文件夹，这个文件夹将用来存放您的博客文件，我单独将e盘作为资料盘，所以我的博客文件在e盘的blog下。

打开创建好的文件夹，在路径处全选输入cmd回车。然后输入npm install hexo-cli -g进行安装hexo。

安装好了以后输入hexo -v检查是否安装成功，如果输出版本号就说明安装成功，你的文件夹里会出现一个blog文件夹。输入hexo init blog初始化文件夹，输入cd blog将目录切换到blog文件夹里，然后输入npm install就会在文件夹里生成一些需要的文件。

配置完成！，接下来输入hexo g生成静态网页，再输入hexo server就可以通过本地服务器打开网站来查看网页

部署Github

这才是我们的目的，让所有人都能看到您的站点。

打开GitHub官网，登录你的账号，如果没有就注册一个。然后新建一个存储库。

输入存储库的名字 格式：youname.github.io比如我的仓库名称为zsuh.github.io，

选择Public公开存储库，记得勾选Add a README file,新建即可。

进入存储库后点击Settings(设置)，左侧点击GitHub Pages(页面)，将Source的Branch改成main点击Save。

hexo部署至GitHub

输入git config --global user.name "你GitHub账号的名字"
例：git config --global user.name "zsuh"

输入git config --global user.email "你GitHub账号的邮箱"
例：git config --global user.email "zsuhao@qq.com"

输入ssh-keygen -t rsa -C "你GitHub账号的邮箱"
例：ssh-keygen -t rsa -C "zsuhao@qq.com"来生成 ssh key。

然后输入cat ~/.ssh/id_rsa.pub输出SSH Key,然后全部复制下来。

打开GitHub点击自己的头像再点击设置，再点击SSH and GPG keys，继续点击New SSH key,输入SSH key的名称，再粘贴刚才辅助的SSH key，点击添加就好了

打开博客目录下的_config.yml我们暂且称呼他为站点配置文件，笔记本、代码编辑器大部分都能打开，推荐使用vscode打开。

搜索deploy修改如下配置

ctrl+s保存即可

在cmd安装插件npm install hexo-deployer-git --save回车后安装GitHub推送插件。

安装好输入hexo c·、hexo g、hexo d上传到GitHub就好了

在地址栏输入你的用户名.github.io即可预览你的博客，在Git Bash Here输入hexo s可以本地预览。
